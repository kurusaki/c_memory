## 動画シナリオ

### メモリの中はどうなっている？

C言語を学んでいると、

- ポインタが難しい
- 配列がよく分からない
- malloc が怖い

と感じる人は多いと思います。

でも実は、これらはすべてメモリの話です。

逆に言えば、
メモリが見えるようになると、C言語の理解は一気に進みます。

今回はその第一歩として、
メモリの中を実際に覗くための
「メモリダンプ関数」
を作ってみましょう。

今回作る関数は、今後のシリーズでも何度も使う共通ツールになります。

まず質問です。

この変数の中身はどこにあるのでしょうか？

```c
int value = 123;
```

多くの人は、「変数に保存されている」と答えます。

もちろん正しいのですが、
実際には、コンピュータのメモリのどこかに保存されています。

今日はその場所と中身を見てみます。


### メモリは巨大な箱の並び


メモリは、たくさんの小さな箱が並んでいるようなものです。

それぞれの箱には、番号（アドレス）が付いています。


```text
                メモリ（たくさんの箱が並んでいる）

+--------+--------+--------+--------+--------+
|        |        |        |        |        |
+--------+--------+--------+--------+--------+
|0x1000  |0x1001  |0x1002  |0x1003  |0x1004  |
+--------+--------+--------+--------+--------+

        1つの箱 = 1バイト
```

1つの箱には、1バイトのデータが保存できます。

この箱に付いている住所のような番号を
アドレスと呼びます。


### アドレスとは？

例えば、こんなイメージです。

```text
1000
1001
1002
1003
1004
```
実際のコンピュータでは、もっと大きな値になります。

例えば、 `0x7ffeedc4a0b8`　のような値です。

これは、メモリ上の場所を表しています。

人間でいう住所のようなものです。


### なぜ16進数を使うの？

アドレスを見ると、
数字だけではなく、
アルファベットも出てきます。

`0x1000`
`0xABCD`
`0xFF`

これは16進数です。


### 16進数とは？

普段私たちが使っているのは10進数です。

`0` ～ `9`、`10`になると桁が上がります。

16進数では、
`0` ～ `9`と
`A` ～ `F`
の16種類の文字を使います。

```
A = 10
B = 11
C = 12
D = 13
E = 14
F = 15
```
となります。



### コンピュータと16進数

コンピュータの内部では、すべて2進数で管理されています。

例えば、2進数で

`11111111`

と表示されると、少し読みにくいですよね。

そこで、

`1111 1111`

のように4ビットごとに区切ると、

```
1111 → F
1111 → F
```

となり、

`FF`

と短く表現できます。

このように、16進数を使うと2進数より短く分かりやすく表現できます。

そのため、メモリやアドレスは16進数で表示されることが多いのです。


### メモリダンプ関数を作ろう

それでは、メモリの中を見るための関数を作ります。

今回はこちらです。

```c
#include <stdio.h>
#include <stddef.h>
#include <ctype.h>

void mdump(void *addr, size_t size)
{
    unsigned char *p = (unsigned char *)addr;

    for (size_t i = 0; i < size; i += 16) {

        printf("%p : ", p + i);

        /* HEX表示 */
        for (size_t j = 0; j < 16; j++) {

            if (i + j < size) {
                printf("%02X ", p[i + j]);
            } else {
                printf("   ");
            }
        }

        printf(" ");

        /* ASCII表示 */
        for (size_t j = 0; j < 16 && i + j < size; j++) {

            unsigned char c = p[i + j];

            if (isprint(c)) {
                printf("%c", c);
            } else {
                printf(".");
            }
        }

        printf("\n");
    }
}

int main(void) {

    int value = 123;
    mdump(&value, sizeof(value));

    return 0;
}
```


今は理解できなくても大丈夫です。

今回は、 「メモリを見る道具」 として使います。

関数の詳しい仕組みは、今後の動画で少しずつ理解していきます。


### 実際にメモリを見てみよう

まずは整数です。

```c
int value = 123;
mdump(&value, sizeof(value));
```

実行すると、こんな結果になります。

```bash
% ./mdump
0x16dda2d98 : 7B 00 00 00                                      {...
```

`123`を保存したメモリの内容を16進数で表示すると、`7B 00 00 00` となっています。

先頭の `7B` は、`123`を16進数で表した値です。


### 文字列も見てみよう

続いて文字列です。

```c
char text[] = "Hello";
mdump(text, sizeof(text));
```

実行すると、こうなります。

```bash
% ./mdump               
0x16fabed94 : 48 65 6C 6C 6F 00                                Hello.
```

これは、それぞれの文字に対応するASCIIコードを16進数で表示したものです。

```text
0x48 → H
0x65 → e
0x6C → l
0x6C → l
0x6F → o
0x00 → ヌル文字（\0）
```

そして最後の `00` は特別な値です。



### NULL文字とは？

```bash
% ./mdump               
0x16fabed94 : 48 65 6C 6C 6F 00                                Hello.
```

この `00` は、文字列の終わりを表しています。

これを NULL（ヌル）文字 と呼びます。

C言語の文字列は、このNULL文字を見つけることで
**「ここで文字列が終わった」** と判断しています。

今後、文字列やポインタを学ぶときに、とても重要な知識になります。


### この関数は今後も使います

今回作った mdump 関数は、今後の動画でも使い続けます。

例えば、

- 配列
- 関数
- 構造体
- ポインタ
- malloc

を学ぶとき、実際にメモリの中を確認していきます。
