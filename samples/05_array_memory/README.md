## 動画シナリオ

配列は、C言語で最もよく使われるデータ構造の一つです。

そして、配列がメモリ上でどのように保存されているのかを理解すると、
添字やアドレス、
さらに今後学ぶポインタも理解しやすくなります。

それでは見ていきましょう。


### 配列を作ってみよう


まずは、5つの整数を持つ配列を用意します。

#### array_memory.c
```c
#include <stdio.h>

int main(void)
{
    int data[5] = {10, 20, 30, 40, 50};

    return 0;
}
```

この配列には、5つの `int` 型のデータが入っています。

では、この5つの値は、メモリ上ではどのように保存されているのでしょうか。


### メモリダンプで確認しよう

以前、「メモリの中はどうなっている？メモリダンプ関数を作ろう」
という動画で、メモリを表示する `mdump` 関数を作成しました。

今回は、このプログラムに `mdump` を追加して、配列の中身を見てみます。

#### array_memory.c
```c
#include <stdio.h>
#include "../mdump.h"

int main(void)
{
    int data[5] = {10, 20, 30, 40, 50};

    mdump(data, sizeof(data));

    return 0;
}
```

`mdump.h` と `mdump.c` は、ひとつ上のディレクトリに配置しています。

コンパイルして実行してみます。

```bash
clang array_memory.c ../mdump.c -o array_memory 
```


```bash
./array_memory
```

【実行結果】
```text
0x16cf9ad70 : 0A 00 00 00 14 00 00 00 1E 00 00 00 28 00 00 00  ............(...
0x16cf9ad80 : 32 00 00 00                                      2...
```

ここで注目してほしいのは、
**4バイトごとに区切ると、1つの `int` 型のデータ**になっていることです。

```text
 0A 00 00 00 → 10
 14 00 00 00 → 20
 1E 00 00 00 → 30
 28 00 00 00 → 40
 32 00 00 00 → 50
```

`int` 型は、多くの環境で4バイトです。

つまり、1つの `int` 型のデータは4バイトで保存されています。

そして、その4バイトのデータが、**連続して並んでいる**ことが分かります。

ここで、以前学んだエンディアンの話も思い出してください。

`10` は16進数では `0A` ですが、メモリ上では `0A 00 00 00` のように保存されています。


### この4バイトごとのデータはどこにある？

では、この4バイトごとのデータは、メモリのどこに保存されているのでしょうか。

ここで登場するのが、**アドレス**です。

アドレスは、メモリ上の場所を表す番号です。


### アドレスを表示してみよう

今度は、先ほどのプログラムに、各要素のアドレスを表示するコードを追加してみます。

#### array_memory.c
```c
#include <stdio.h>
#include "../mdump.h"

int main(void)
{
    int data[5] = {10, 20, 30, 40, 50};

    mdump(data, sizeof(data));

    printf("&data[0] = %p\n", &data[0]);
    printf("&data[1] = %p\n", &data[1]);
    printf("&data[2] = %p\n", &data[2]);
    printf("&data[3] = %p\n", &data[3]);
    printf("&data[4] = %p\n", &data[4]);

    return 0;
}
```

ここで出てきた `&` は、**変数のアドレスを取得する演算子**です。

`&data[0]`は、`data[0]`が保存されている場所、つまりアドレスを取得しています。

`&` については、今後の動画で詳しく説明します。

コンパイルして実行してみましょう。

```text
0x16d876d70 : 0A 00 00 00 14 00 00 00 1E 00 00 00 28 00 00 00  ............(...
0x16d876d80 : 32 00 00 00                                      2...
&data[0] = 0x16d876d70
&data[1] = 0x16d876d74
&data[2] = 0x16d876d78
&data[3] = 0x16d876d7c
&data[4] = 0x16d876d80
```

※ アドレスは実行するたびに変わります。
ここでは、実際に実行した結果を掲載しています。


先ほどの mdump のアドレスと見比べると、

`data[0]` は `0x16d876d70`、

`data[1]` は `0x16d876d74` に保存されていることが分かります。

また、アドレスが 4バイトずつ増えていることにも注目してください。

これは、`int` 型が4バイトだからです。


### 考えてみよう

ここで問題です。

`&data[2]` のアドレスが `0x16d876d78` だとすると、

`&data[3]` のアドレスは、いくつになるでしょうか？

少し考えてみてください。

答えは、`0x16d876d7c` です。

4バイト分進むため、次の要素のアドレスも4バイト先になります。

この規則性が、配列の特徴です。


### 添字とアドレスの関係

ここで、添字とアドレスを整理してみましょう。

| 添字 | アドレス | 値 |
| :----------: | :------------: | :-----: |
| data[0]| 0x16d876d70 | 10 |
| data[1]| 0x16d876d74 | 20 |
| data[2]| 0x16d876d78 | 30 |
| data[3]| 0x16d876d7c | 40 |
| data[4]| 0x16d876d80 | 50 |


`data[0]` や `data[1]` の数字を、**添字（そえじ）**と呼びます。

添字は、**連続したメモリ領域の何番目の要素なのか**を表しています。

つまり、

`data[0]` は、先頭

`data[1]` は、その次

`data[2]` は、さらにその次という意味になります。

このように、配列では、**添字・アドレス・値**が規則的に対応しているのです。


### まとめ

今回は、配列がメモリ上でどのように保存されているのかを見てきました。

ポイントは4つです。

- 配列は連続したメモリ領域に保存される
- `int` 型は多くの環境で4バイト
- `&` を付けるとアドレスを取得できる
- 添字は連続したメモリ領域の何番目の要素かを表している

配列が連続したメモリ領域であることが分かると、

**「配列名は先頭要素を指す」**

というポインタの考え方も理解しやすくなります。
