## 動画シナリオ

前回は、「ポインタはアドレスを保存する変数」という基本を学びました。

今回は、ポインタを使っていると必ず登場する**ポインタ演算**について学びます。

例えば、`p + 1`というコードです。

1を足しているので、1バイト進みそうに見えます。

しかし実際には、`int`型のポインタなら4バイト進みます。

なぜ4バイト進むのでしょうか？

今回は、メモリを見ながらその仕組みを理解していきましょう。


### サンプルコード①

まずは、今日最初のサンプルコードです。

#### 01_pointer.c
```c
#include <stdio.h>

int main(void)
{
    int array[] = {10, 20, 30, 40, 50};

    int *p = array;

    printf("sizeof(int) = %zu\n", sizeof(int));

    printf("p     = %p\n", (void *)p);
    printf("p + 1 = %p\n", (void *)(p + 1));
    printf("p + 2 = %p\n", (void *)(p + 2));
    printf("p + 3 = %p\n", (void *)(p + 3));
    printf("p + 4 = %p\n", (void *)(p + 4));

    return 0;
}
```

### 実行

コンパイルして実行してみましょう。

```bash
clang 01_pointer.c -o 01_pointer
```

```bash
./01_pointer
```

【実行結果】
```text
sizeof(int) = 4
p     = 0x16fc0ad50
p + 1 = 0x16fc0ad54
p + 2 = 0x16fc0ad58
p + 3 = 0x16fc0ad5c
p + 4 = 0x16fc0ad60
```

実行結果はこのようになります。

実際のアドレスは環境によって変わります。

注目してほしいのは、4バイトずつ増えていることです。

### 配列は連続したメモリ

まず、配列はメモリ上にこのように並んでいました。

| アドレス | 値 |
| :----------: | :------: |
| 0x16fc0ad50 | 10 |
| 0x16fc0ad54 | 20 |
| 0x16fc0ad58 | 30 |
| 0x16fc0ad5c | 40 |
| 0x16fc0ad60 | 50 |

`int`型は4バイトなので、隣り合う要素の先頭アドレスは4バイトずつ離れています。

### `p`はどこを指している？

```c
int *p = array;
```

このような式の中では、配列名`array`は、配列の先頭要素のアドレスとして扱われます。

そのため、`p`には配列の先頭要素`array[0]`のアドレスが入ります。

つまり、`p`は`array[0]`を指しています。


### `p + 1`とは？

では、`p + 1`は、どういう意味でしょう？

`0x16fc0ad50`に1を足しているので、`0x16fc0ad51`になりそうですが、実際には`0x16fc0ad54`になります。

これは、ポインタ演算では**1バイト進むのではなく、型1個分進む**というルールだからです。

ポインタ`p`に`n`を足すと、アドレスは次の分だけ進みます。

```text
n × ポインタが指す型のサイズ
```

今回の`p`は`int *`で、`sizeof(int)`は4バイトです。

そのため、`p + 1`では、

```text
1 × 4バイト = 4バイト
```

進みます。

同じように、`p + 2`では8バイト、`p + 3`では12バイト進みます。

ここで注意したいのは、`p + 1`と書いただけでは、`p`自身に保存されているアドレスは変わらないことです。

`p + 1`は、`p`から型1個分進んだアドレスを表しています。


### 型によって進む距離が変わる

今回のMac環境では、それぞれ次のように進みます。

- `char *`なら1バイト
- `short *`なら2バイト
- `int *`なら4バイト
- `long *`なら8バイト

ただし、型のサイズは実行環境によって異なる場合があります。

なお、`sizeof(char)`の値は、C言語の規格上、常に1です。

実際に何バイト進むかは、`sizeof`を使って確認できます。

つまり、ポインタ演算では**ポインタの型**がとても重要になります。


### ポインタを順番に進めるには？

ここまで、`p + 1`という書き方でポインタ演算の仕組みを確認しました。

配列をポインタで順番に処理するときには、`p++`という書き方も使われます。

`p++`を実行すると、`p`自身に保存されているアドレスが、次の要素のアドレスへ変わります。


### サンプルコード②

こちらが2つ目のサンプルです。

添字を使ったループとポインタを使ったループを比較してみます。

#### 02_pointer_loop.c
```c
#include <stdio.h>

int main(void)
{
    int array[] = {10, 20, 30, 40, 50};

    printf("添字を使ったループ\n");

    for (int i = 0; i < 5; i++) {
        printf("%d\n", array[i]);
    }

    printf("\nポインタを使ったループ\n");

    int *p = array;

    for (int i = 0; i < 5; i++) {
        printf("%p : %d\n", (void *)p, *p);
        p++;
    }

    return 0;
}
```

添字を使ったループを見てください。
```c
for (int i = 0; i < 5; i++) {
    printf("%d\n", array[i]);
}
```
これは、普段よく使う配列の書き方ですね。

続いて、ポインタを使ったループです。
```c
int *p = array;

for (int i = 0; i < 5; i++) {
    printf("%p : %d\n", (void *)p, *p);
    p++;
}
```

### 実行

コンパイルして実行してみます。

```bash
clang 02_pointer_loop.c -o 02_pointer_loop  
```

```bash
./02_pointer_loop
```

【実行結果】
```text
添字を使ったループ
10
20
30
40
50

ポインタを使ったループ
0x16f5c6d40 : 10
0x16f5c6d44 : 20
0x16f5c6d48 : 30
0x16f5c6d4c : 40
0x16f5c6d50 : 50
```

添字を使ったループでは、`10`から`50`まで表示されます。

ポインタを使ったループでは、`p++`するたびに、`p`が4バイトずつ進んでいることが分かります。

そのため、`*p`で取り出される値も、`10`、`20`、`30`、`40`、`50`と順番に変わります。

### 添字とポインタを比較する

#### 添字を使ったループ
```c
for (int i = 0; i < 5; i++) {
    printf("%d\n", array[i]);
}
```

#### ポインタを使ったループ
```c
int *p = array;

for (int i = 0; i < 5; i++) {
    printf("%p : %d\n", (void *)p, *p);
    p++;
}
```

書き方は違いますが、どちらも配列の先頭から最後まで順番に値を取り出しています。

つまり、配列は添字だけでなく、ポインタでも扱えるということです。

実は、添字を使った`array[i]`という書き方は、

ポインタを使うと`*(array + i)`と書くことができます。

なぜこの2つが、同じ意味になるのでしょうか？

そこには、配列とポインタの深い関係があります。

この続きは、次回詳しく見ていきましょう。

### まとめ

今回は、ポインタ演算について学びました。

今回のポイントは、次の4つです。

- `p + 1`は1バイトではなく、ポインタが指す型1個分進む
- 今回の環境では、`int *`は4バイトずつ進む
- `p++`を使うと、ポインタを次の要素へ進められる
- 添字を使ったループも、ポインタを使ったループも、配列を先頭から順番に読み取っている

ポインタ演算では、ポインタの型と、その型のサイズが重要です。

そして、添字を使った`array[i]`という書き方は、ポインタを使うと`*(array + i)`と表せます。

なぜ、この2つが同じ意味になるのでしょうか？

そこには、配列とポインタの深い関係があります。
