## 動画シナリオ

前回は、
```c
int   i = 20;
```
をメモリダンプすると、`14 00 00 00` になることを確認しました。

`20` が `14` になる理由は、16進数で表示しているからでした。

でも、もう一つ疑問が残っています。

なぜ、`14 00 00 00` なのでしょうか？

普通に考えると、`00 00 00 14` になりそうですよね。

今回は、その理由となる **エンディアン** について学んでいきます。


### エンディアンとは？

エンディアンとは、複数バイトのデータをメモリに並べる順番のことです。

この並べ方には2種類あります。

- リトルエンディアン
- ビッグエンディアン

まずは、分かりやすい例で見てみましょう。


### 例：0x12345678

例えば、`0x12345678` という4バイトのデータがあるとします。

このデータをメモリに保存するとき、エンディアンによって並び方が変わります。


### リトルエンディアン

まずは、リトルエンディアンです。

リトルエンディアンでは、

`0x12345678` → `78 56 34 12`

という順番で保存されます。

つまり、**下位バイトから保存する方式** です。

現在のMac（Apple SiliconやIntel Mac）、Windowsでは、リトルエンディアンが採用されています。


### ビッグエンディアン

次は、ビッグエンディアンです。

`0x12345678` → `12 34 56 78`

となります。

こちらは、**上位バイトから保存する方式** です。

こうして比較すると、並び方がちょうど逆になっていることが分かります。


### 前回の「20」を見てみよう

では、前回の
```c
int   i = 20;
```
に戻ってみましょう。

20を16進数で表すと、`0x00000014` になります。

`0x00000014` → `14 00 00 00`

となります。

前回見たメモリダンプは、**リトルエンディアンだから、この順番だった**というわけです。

### なぜ2種類あるの？

では、なぜ2種類あるのでしょうか。

これは、CPUのアーキテクチャによって採用されている方式が異なるためです。

どちらが正しいというわけではありません。

それぞれに歴史や設計上の理由があります。

現在のパソコンでは、リトルエンディアンが主流です。

ネットワークでは、異なるコンピュータ同士が同じ並び方で通信できるように、ビッグエンディアンが標準として使われています。

これを、**ネットワークバイトオーダー**と呼びます。


### 実際に確認してみよう

前回のプログラムの実行結果を、もう一度見てみましょう。

```text
===== char  (20) : 1バイト =====
0x16efb2d7b : 14                                               .
===== short (20) : 2バイト =====
0x16efb2d78 : 14 00                                            ..
===== int   (20) : 4バイト =====
0x16efb2d74 : 14 00 00 00                                      ....
===== long  (20) : 8バイト =====
0x16efb2d68 : 14 00 00 00 00 00 00 00                          ........
```

この結果を見るだけで、
このMacが **リトルエンディアン** でデータを保存していることが分かります。

メモリダンプを見ることで、CPUがデータをどのような順番でメモリに保存しているのかまで分かるんですね。


### まとめ

今回は、エンディアンについて学びました。

今回のポイントです。

- エンディアンはメモリに並べる順番のこと
- リトルエンディアンは下位バイトから保存する
- ビッグエンディアンは上位バイトから保存する
- MacやWindowsではリトルエンディアンが主流
- ネットワーク通信ではビッグエンディアンが使われる



前回の `14 00 00 00` という並びにも、ちゃんと理由がありました。
