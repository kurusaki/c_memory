## 動画シナリオ


前回は、メモリダンプ関数を作りました。
これからのシリーズでは、この関数を使って実際にメモリの中を見ながら学んでいきます。

でも、その前に一つ確認しておきたいことがあります。

そもそも、コンピュータのメモリとは何なのでしょうか？

メモリの仕組みを理解すると、
ポインタや配列、構造体など、
C言語で難しいと言われる内容がぐっと理解しやすくなります。

今回は、

- メモリとは何か
- アドレスとは何か
- バイトとは何か

を、実際にC言語を書きながら確認していきます。


### メモリとは？

まずは、次のプログラムを書いてみます。

#### sample1.c
```c
#include <stdio.h>

int main(void)
{
	int age = 20;

	printf("%d\n", age);
	
	return 0;
}
```

コンパイルして実行してみます。
```bash
clang sample1.c -o sample1
```

```bash
./sample1
```

実行結果
```bash
20
```

20と表示されました。

この20という値は、一体どこに保存されているのでしょうか？

実は、この値はメモリに保存されています。

メモリとは、プログラムが実行中にデータを保存する場所です。

変数を作るということは、
メモリのどこかに保存場所を確保することなんです。



### メモリは1バイトの箱が並んでいる

メモリは、一つの大きな箱ではありません。
```
┌──────────────────────────────────────────────┐
│                  メモリ                      │
└──────────────────────────────────────────────┘
```

実際には、1バイトごとの小さな箱が横に並んでいます。
```
┌────┬────┬────┬────┬────┬────┬────┬────┐
│1B  │1B  │1B  │1B  │1B  │1B  │1B  │1B  │
└────┴────┴────┴────┴────┴────┴────┴────┘
```

それぞれの箱には1バイト（8ビット）のデータを保存できます。

変数を作ると、この箱の一部が使われます。

まずは、
「メモリは小さな箱がたくさん並んでいる」
というイメージを持ってください。



### アドレスとは？

では、たくさん箱が並んでいる中で、
目的の箱はどうやって見つけるのでしょうか？

そこで登場するのが**アドレス**です。

アドレスとは、
メモリの箱一つひとつにつけられた住所です。


例えば、この左側がアドレスです。

メモリは、このような表のイメージで考えることができます。

```
アドレスとデータ
┌──────────┬────────┐
│ アドレス │ データ │
├──────────┼────────┤
│ 0x1000   │   ??   │
│ 0x1001   │   ??   │
│ 0x1002   │   ??   │
│ 0x1003   │   ??   │
└──────────┴────────┘
```

メモリは1バイト単位で管理されているので、
1バイト進むごとに、
`0x1000`
`0x1001`
`0x1002`
というように、アドレスが1ずつ増えていきます。

この `0x` は、16進数であることを表しています。

つまり、アドレスは通常16進数で表現されます。


### C言語でアドレスを見てみよう

実際に、変数のアドレスを表示してみます。

#### sample2.c
```c
#include <stdio.h>

int main(void) 
{
    int age = 20;

    printf("age = %d\n", age);
    printf("address = %p\n", (void *)&age);

    return 0;
}
```

コンパイルして実行してみます。
```bash
clang sample2.c -o sample2
```

```bash
./sample2
```

実行結果
```bash
age = 20
address = 0x16d01ed68
```

実行すると、`age`は20、`address`は16進数で表示されます。

この値は、変数 `age` が保存されているメモリ上のアドレスです。

環境や実行するたびに値が変わることがありますが、それは正常です。

ここでは、変数には住所があるということだけ覚えてください。

### バイトとは？

では、メモリの箱一つには、どれくらいのデータが入るのでしょうか？

それが1バイトです。

1バイトは、8ビットでできています。

```text
01010101
↑↑↑↑↑↑↑↑
8個の0と1（8ビット）
```

この8個の0と1で、1バイトになります。

つまり、メモリの箱一つの大きさが、1バイトということです。


### 型によって使う大きさが違う

では、変数は何個の箱を使うのでしょうか？

C言語には、sizeofという演算子があります。

実際に確認してみます。

#### sample3.c
```c
#include <stdio.h>

int main(void)
{
    printf("char : %zu\n", sizeof(char));
    printf("short : %zu\n", sizeof(short));
    printf("int : %zu\n", sizeof(int));
    printf("long : %zu\n", sizeof(long));
    printf("double : %zu\n", sizeof(double));

    return 0;
}
```

コンパイルして実行してみます。
```bash
clang sample3.c -o sample3
```

```bash
./sample3
```

実行結果

Macでは、次のようになります。
```bash
char : 1
short : 2
int : 4
long : 8
double : 8
```

これは、それぞれの型が使うバイト数です。


#### データ型が使うメモリ（1バイトの箱）


### int型は4バイト使う

先ほどの 

```c
int age = 20;
``` 

では、Macでは4バイト分のメモリが使われます。


この4つの箱が、変数 age のために使われます。

この中に20という値が保存されます。


```text
アドレス     データ
0x1000       ??
0x1001       ??
0x1002       ??
0x1003       ??
    ↑
この4バイトに20が保存される
```

次回は、前回作ったメモリダンプ関数を使って、
この4バイトの中に実際にどのようなデータが保存されているのかを見ていきます。


### まとめ

今回は、コンピュータのメモリの基本を学びました。

- メモリはデータを保存する場所
- メモリは1バイトごとの箱が並んでいる
- アドレスは箱の住所
- 型によって使うバイト数が違う
- 変数はメモリ上に保存される


ここまで理解できれば、
今後学ぶ配列やポインタも理解しやすくなります。
